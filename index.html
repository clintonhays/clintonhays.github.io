Newsletter Signup Page

This is a signup page for a newletter focusing on forms and inputs.

Newsletter Signup

Parameters:

Create a form page from a client mockup using HTML and CSS.

Challenges:

HTML forms are very robust, requiring many attributes and labels. It was important to make sure that everything was connected correctly. Additionally, inputs are not as easy to style as other elements. It took a lot of research to find out what parts of inputs can be styled. I attempted to use a neumorphism approach to the style. It was challenging and rewarding, but I'm sure that I can refactor the design to be even more dynamic. I plan to add validation to this form using Javascript.

Takeaways:

Forms are something that I want to learn more about. I want to make sure that in the future my forms are accessible and use correct markup. In addition, when styling forms, I realized that user experience and accessibility need to be at the forefront of consideration. It was a great learning experience and overall quite a fun project.

Photo Gallery

This is a simple interactive photo gallery with lightbox and search functions.

Photo Gallery

Parameters:

Create an interactive photo gallery that allows search and uses lightbox to interact with photos.

Challenges:

There were two main challenges with this project. The first was understanding how to properly use libraries, plugins, and modules. I had some initial trouble getting the baguettebox plugin working becuase I wasn't sure how the plugin interacted with my own code, so the project file structure needed some adjusting on my end. After learning that a plugin really is just that...it all made sense. The second challenge was creating the search function from scratch. It took some research, but I eventually found a tutorial that really made sense. This was one of the first in-depth Javascript projects I attemtped. After learning more about DOM traversal, search functions are much easier to understand and implement.

This project showed me that plugins and libraries are powerful and useful tools. Since completing this project, I haven't implemented nearly enough of them into my other projects. Since I'm still learning, I like to use as much vanilla JS as possible, but I am ready to start expanding into the world of libraries and plugins.

Four-in-a-Row Game

This is a simple four in a row game focusing on object oriented programming with vanilla JS.

Four-in-a-Row

Parameters:

Create a multiplayer, connect 4 style game focusing on OOJS.

Challenges:

This was hands down the most JavaScript heavy project I tackled in the beginning of my coding journey. My main hurdles included getting a grasp on how objects interact with one another. Trying to keep the relationships of the objects straight lead to some very frustrating moments. I also had a couple of issues with the file structure. Up to this point I had been working with a single JS file that I could peruse with ease when needed. This project really expanded the project file tree. But, I was eventually able to wrap my head around both of these aspects.

Takeaways:

Object oriented JavaScript is a really diverse and powerful tool. There is a lot of structure that belies just how flexible the ability to code this was really is. While it can seem at times like a spiderweb of interacting objects, there is obviously, and necessarily, an order to the seeming chaos. Personally, I really like working with objects and arrays, so I find this kind of programming particularly rewarding. This project really solidified that for me.

React Scoreboard

This is a scoreboard with multiple functioning parts including a stopwatch and leader marker.

React Scoreboard

Parameters:

Create a scoreboard using React and focusing on components and state.

Challenges:

React really is a beast of a framework. The myriad challenges posed by React as a new developer was a bit overwhelming at first. I went through this particular tutorial twice and will probably go through one more time just to make sure I have all the concepts down. The main challenge for me is understanding how to structure an app from the beginning. I need more practice in looking at an app or mockup and breaking it down mentally into individual components. I know this will come with more practice, but as of right now, it’s still quite the challenge. The syntax and component interaction is still a bit muddled for me, but it’s becoming clearer each time I work with React. Another minor issue I ran into was making sure my file structure was correct, especially since I wanted to deploy through Github pages. Fortunately, there is plenty of documentation out there with how to deploy on Github pages.

Takeaways:

React will absolutely be my go to for frontend development. Although it’s a bit hard to understand at the moment, the theory behind it makes a lot of sense to me. I’ve always tried to break things down to their base elements in most things I’ve built. I like that React is built around exactly that. With smaller components in an app, it is easier to manipulate and improve over time. I need more practice with this framework, but I find it really fun to work with.

Random Quote Machine

Parameters:

Create an app based on user stories that returns a random quote.

Challenges:

It had been a bit since I used an API and I had never integrated one into a React app before. So, my main challenge was understanding that in a React app the API could be used to update state. This made it much easier to implement the API data into the app. Second, I was a little unsure how CSS transitions worked with React. Looking back now, I realize that this shouldn’t have been a challenge at all, but while making the app, it didn’t occur to me that I could just target the property being set in the app through my SASS file directly. This is definitely one of those learning moments that will make things a lot easier moving forward.

Takeaways:

This project brought together two of my favorite things about web development...React and APIs. I had a lot of fun building this project and it has really inspired me to move forward with making an API database so I can create a fullstack app of my own.
