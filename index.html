Conference Demo Site

Parameters:

Create a site based on a client mockup using the Bootstrap framework.

Challenges:

Bootstrap was a new framework for me, I had trouble getting my head around the grid system in the beginning. Bootstrap is also very limiting, so adjusting small details or style choices is almost impossible.

Takeaways:

Newsletter Signup Page

This is a signup page for a newletter focusing on forms and inputs.

Newsletter Signup

Parameters:

Create a form page from a mockup using HTML and CSS.

Challenges:

HTML forms are very robust, requiring attributes and labels. It was a challenge to make sure that everything was connected correctly. Additionally, inputs are not as easy to style as other elements. It took a lot of research to find out what parts of inputs can be styled. I attempted to use neumorphism approach to the style. It was challenging and rewarding, but I'm sure that I can refactor the design to be even more dynamic. I plan to add validation to this form using Javascript.

Takeaways:

Forms are something that I want to learn more about. I want to make sure that in the future my forms are accessible and use correct markup. In addition, when styling forms, I realized that user experience needs to be at the forefront of consideration. It was a great learning experience and overall quite a fun project.

Employee Directory

This is a directory that uses an API to generate random employee information.

Employee Directory

Parameters:

Create an interactive directory that displays information cards and detailed information modals.

Challenges:

This was my first attempt at using an API. Thankfully, using promises made it a lot easier. My biggest speedbump came when trying to wrap my head aorund pulling information from the API. Once I started thinkning in terms of objects, it made much more sense. I also ran into a little trouble when creating the search function because I didn't fully understand the difference between static and live html lists. Now that I have a better understanding of these lists, I'll hopefully run into fewer issues in the future.

Takeaways:

Almost immediately after working on this project, I started to think of other projects that I could use an API for. I've got a few fun things I'd like to do just get some more practice in. There is something very interesting to me about pulling data out of collections of objects. I am very interested in the practices of consolodating, effeciency, and using the right tool for the job. I think APIs offer a great way hit those key areas.
