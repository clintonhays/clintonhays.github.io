Newsletter Signup Page

This is a signup page for a newletter focusing on forms and inputs.

Newsletter Signup

Parameters:

Create a form page from a mockup using HTML and CSS.

Challenges:

HTML forms are very robust, requiring many attributes and labels. It was important to make sure that everything was connected correctly. Additionally, inputs are not as easy to style as other elements. It took a lot of research to find out what parts of inputs can be styled. I attempted to use a neumorphism approach to the style. It was challenging and rewarding, but I'm sure that I can refactor the design to be even more dynamic. I plan to add validation to this form using Javascript.

Takeaways:

Forms are something that I want to learn more about. I want to make sure that in the future my forms are accessible and use correct markup. In addition, when styling forms, I realized that user experience needs to be at the forefront of consideration. It was a great learning experience and overall quite a fun project.

Employee Directory

This is a directory that uses an API to generate random employee information.

Employee Directory

Parameters:

Create an interactive directory that displays information cards and detailed information modals.

Challenges:

This was my first attempt at using an API. Thankfully, using promises made it a lot easier. My biggest speedbump came when trying to wrap my head aorund pulling information from the API. Once I started thinkning in terms of objects, it made much more sense. I also ran into a little trouble when creating the search function because I didn't fully understand the difference between static and live html lists. Now that I have a better understanding of these lists, I'll hopefully run into fewer issues in the future.

Takeaways:

Almost immediately after working on this project, I started to think of other projects that I could use an API for. I've got a few fun things I'd like to do just get some more practice in. There is something very interesting to me about pulling data out of collections of objects. I am very interested in the practices of consolodating, effeciency, and using the right tool for the job. I think APIs offer a great way hit those key areas.

Photo Gallery

This is a simple interactive photo gallery with lightbox and search functions.

Photo Gallery

Parameters:

Create an interactive photo gallery that allows search and uses lightbox to interact with photos.

Challenges:

There were two main challnges with this project. The first was understanding how properly uss libraries, plugins, and modules. I had some initial trouble getting the baguettebox plug in working becuase I wasn't sure how the plugin interacted with my own code so the project file structure needed some adjusting on my end. After learning that a plugin really is just that...it all made sense. The second challenge was creating the search function from scratch. It took some research, but I eventually found a tutorial that really made sense. This was one of the first in depth Javascript projects I attemtped. After learning more about DOM traversal, search functions are much easier to understand and implement.

Takeaways:

This project showed me that plugins and libraries are powerfula nd useful tools. Since completing this project, I haven't implemented nearly enough of them into my other projects. Since I'm still learning, I like to use as much vanilla JS as possible, but I am ready to start expanding into the world of libraries and plugins.
